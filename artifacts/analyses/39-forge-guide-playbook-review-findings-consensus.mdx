## Problem

PR #39 (`feat(forge-guide): adopt multi-tab guide playbook learnings`) is doc-only, +234/-15 across 4 files, bumps skill `0.3.0→0.3.1` and plugin `0.7.1→0.7.2`. It distills a real production session (16 chapters / 29 SVGs) into the skill as 5 new phases (2 §4, 2.5, 3.5, 5), a CSS defaults table, anti-patterns 8→23, cluster specialization, and a SVG path warning. CI passes.

Initial review surfaced 10 findings across 3 severity tiers. Question: which fix-before-merge, which fix-in-follow-up, which drop?

| α | Focus |

## Consensus ρ

**Fix-then-merge** — 4 surgical fixes block merge; 4 land as follow-up; 2 drop.

### Rationale

All 3 panelists agreed on the **fix-then-merge** verdict — no one wanted ship-as-is, no one wanted split, no one wanted defer. The disagreements were on **tier assignment**, not on the validity of findings.

For 2-1 splits, the consensus biased toward the majority WHEN the fix was cheap (#1, #2, #3, #4) — the dissenter typically conceded the concern but disagreed on timing. For the 3-way split (#6), the architect's drift-risk argument and product-lead's "intentional context separation" argument were both valid — the mediator hybrid (DUPE marker comment) addresses both without forcing a choice.

### Trade-offs

1. **Author momentum vs. polish gate:** institutional memory is fresh now (session just ended); delaying merge risks losing context for follow-ups. But shipping #1 unfixed means every first-time user trips on the report-before-QA inversion. The 4 fix-before items are <60 min of work — the gate is cheap.
2. **Linux-first author vs. cross-platform marketplace:** product-lead noted the playbook itself never qualified the platform; this is a pre-existing gap. Architect + devops countered that the SKILL.md going to public marketplace makes silent macOS/Windows failures a credibility issue. Verdict erred on the side of cross-platform — fix is 2-3 lines.
3. **SKILL.md size (548 lines) vs. reference delegation:** most new content went into SKILL.md instead of `references/`. Architect flagged this as drift surface growing. Not blocking, but next major bump should consider extracting Phase 3.5/5 to `references/phase-qa.md`.

## Implementation Notes

### Fix-before-merge changes (target: ≤60 min)

**SKILL.md edits in `plugins/forge/skills/forge-guide/SKILL.md`:**

1. **Phase reorder (#1, resolves #7):**
   - Move current `## Phase 5 — Browser QA` block (lines 450-506) to before `## Phase 4 — Report`
   - Renumber: Phase 5 Browser QA → Phase 4, current Phase 4 Report → Phase 5
   - OR: keep numbering, rename current Phase 4 to "Phase 6 — Report & Deploy" and move it last
   - Update Phase 3.5 Step 5 wording ("before Phase 4" → "before deploy")

2. **Cross-platform PNG (#2, bundles #8):**
   - In Phase 2.5 Step 3 block, replace single Flatpak command with platform table:
     ```
     Linux (Flatpak):  flatpak --user run --branch=49 ... rsvg-convert ...
     Linux (apt):      apt install librsvg2-bin && rsvg-convert -z 2 -o out.png in.svg
     macOS:            brew install librsvg && rsvg-convert -z 2 -o out.png in.svg
     Windows:          wsl rsvg-convert ...  (or skip PNG QA → rely on Phase 5 browser QA)
     ```
   - In Phase 3.5 Step 1, add: "Requires `rsvg-convert` — see Phase 2.5 Step 3 for install."

3. **Server orphan (#3):**
   - In Phase 5 Step 1, replace:
     ```bash
     cd {ROOT} && python3 -m http.server 8080 &
     ```
     with:
     ```bash
     cd {ROOT} && python3 -m http.server ${PORT:-8090} &
     SERVER_PID=$!
     trap "kill $SERVER_PID 2>/dev/null" EXIT
     ```
   - Default port 8090 (less collision-prone than 8080).

4. **`.gris` quote (#4):**
   - In Anti-Patterns Diagram-level table, change `gris muted applied to active steps` → `\`.gris\` muted applied to active steps` (backticks signal CSS class).
