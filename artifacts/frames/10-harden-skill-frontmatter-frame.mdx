## Problem

Forge's 5 SKILL.md files currently use inline `description:` fields with `|`-separated trigger phrases. They parse correctly today, but the format is fragile: a single colon inside a future trigger phrase would silently break the YAML frontmatter in the Claude Code skills CLI, Codex's agent picker, and similar tooling. The `fireworks-tech-graph` project was hit by exactly this — three community PRs landed in four minutes trying to fix it, and only the third (`>-` block scalar) survived.

Two adjacent issues compound the first:
1. No regression anchors exist for high-drift skills (`forge-chart`, `forge-gallery`) — output quality can silently regress between skill versions with no signal.
2. `forge-chart` and `forge-guide` SKILL.md files are each ~23 KB and still carry inline tables/decision-trees that belong in `references/`.

## Who

- **Primary:** Mickael (forge maintainer) — future edits to triggers shouldn't require YAML debugging; skill quality shouldn't silently regress across versions.
- **Secondary:** End users of the forge plugin — install breakage on colon-containing triggers is an invisible failure mode until someone reports a skill "not being found."

## Constraints

- Must keep all 5 skills callable by their current triggers (no semantic change to `description:` content).
- Manifest generator (`scripts/gen-plugin-manifest.py`) and the `manifest` pre-commit hook must continue to parse frontmatter without changes — whatever YAML form we pick has to round-trip through the existing tooling.
- Fixture format must be cheap to author and diff-friendly (JSON preferred; stable ordering).
- Compression targets (<12 KB for `forge-chart` / `forge-guide`) are a goal, not a hard limit — don't degrade clarity to hit a byte count.

## Out of Scope

- Changes to the skills' actual behavior or output format.
- Cross-skill refactors (e.g. merging `references/` across skills).
- Adopting fireworks-tech-graph's broader patterns (Python List Method mandate, agents/openai.yaml, etc.).
- Any non-forge plugin in this marketplace.
