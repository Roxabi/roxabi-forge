## Problem

Two recent audits (`2026-04-12-competitor-skills-analysis.md` + `2026-04-14-gmdiagram-delta-analysis.md`) identified high-leverage, low-risk patterns from three competitor skills (`architecture-diagram-generator`, `fireworks-tech-graph`, `gmdiagram`) that forge does not yet have. Each item is documented, sourced, and scoped to a specific file. None of them requires new dependencies or breaks existing output.

Without them, `forge-chart` ships without a Deliver-phase pre-flight check, fgraph has no semantic color tokens or boundary/cluster primitive, there is no SVG validator gate, and authors have no AI-domain primer or pixel-exact layout constants to prevent silent drift in layered diagrams.

This issue bundles all seven Tier-1 lifts into a single sprint, mirroring the closed #5 Tier-1 pattern. It is the "Issue A" bundle proposed at the end of the 2026-04-14 delta analysis.

## Who

- **Primary:** forge skill authors (`forge-chart` in particular) — gain a checkable Deliver gate, semantic tokens, boundary groups, SVG validator, layout constants, and a domain-patterns primer.
- **Secondary:** downstream artifact consumers (gallery/doc generators) — benefit from stricter SVG output and more expressive architecture diagrams (cluster frames, semantic colors).

## Constraints

- No new runtime dependencies; `validate-svg.sh` is shell-only (uses tools already available: `xmllint` / `rsvg-convert` if present, graceful fallback otherwise).
- No breaking changes to existing outputs — all additions are additive (new CSS variables, new selectors, new SKILL.md sections).
- Must not overlap with #10 (fireworks frontmatter/fixtures/compression) or #11 (forge-slides polish).
- Must not re-open the deferred items from closed #7 (auto-layout, JSON IR two-step).
- Touches three sub-trees: `plugins/forge/skills/forge-chart/`, `plugins/forge/references/` (tokens, components.css, graph-templates, shape-vocabulary), `plugins/forge/scripts/`. Multi-file but single-domain (forge plugin).

## Out of Scope

- New diagram types (Gantt, UML, ER, pie, radar, funnel, bubble, scatter) — tracked as future "Issue B".
- PNG/PDF export via `export.sh` — tracked as future "Issue C".
- Periodic audit report template (D5) — deferred until the first real audit is run.
- Auto-layout / Python-generated diagrams + JSON IR two-step pipeline — deferred in closed #7.

**Tier: F-lite** — clear scope, single domain (forge plugin), sources cited per task, no unknowns. Spans ~7 small edits across skills + references + scripts, so a short spec + plan is worth the de-risking time.
