## Problem

Forge's artifact catalog covers guides, epics, charts, galleries, and init — but has no primitive for magazine-quality presentation decks. The 2026-04-12 competitor analysis (`artifacts/analyses/2026-04-12-competitor-skills-analysis.md`) identified that Visual Explainer is the only reference repo of the three with a slide-deck generator, leaving forge unable to produce shareable deck-style artifacts (brand pitches, project walkthroughs, design reviews) without hand-rolling HTML.

The gap becomes visible every time a forge user wants to present an epic or architecture doc as a navigable deck rather than a scrolling page. Current options are: produce a scrolling guide (wrong format), export to external tools like Keynote/Slides (breaks the single-file `file://` workflow and loses aesthetic tokens), or hand-write slide HTML from scratch.

A 6th skill `forge-slides` closes the gap, keeping decks in the same self-contained HTML pipeline as the other 5 artifact types.

## Who

- **Primary:** Mickael, authoring presentation decks via the forge pipeline (brand iterations, project reviews, epic walkthroughs).
- **Secondary:** downstream viewers receiving a single-file deck over Telegram/Discord/Drive — must open offline on any device without a server.

## Constraints

- Must work offline under `file://` — no server-side rendering, no local http dep.
- External deps limited to CDN fonts + Mermaid CDN (matches existing forge artifacts).
- Reuse existing primitives: 6 aesthetic files (`aesthetics/*.css`), `fgraph-base.css` for in-slide diagrams, `base/mermaid-init.js` for diagram init.
- Follow the Forge 4-phase pipeline (Context → Decision → Generation → Deliver), matching the other 5 skills' UX.
- Output path: `~/.roxabi/forge/<project>/slides/{name}.html`.
- Respect `prefers-reduced-motion` for transitions.
- Keyboard navigation is mandatory (arrows, space, home/end).

## Out of Scope

- PDF export via headless Chrome — deferred beyond v0.1.
- Slide builder UI / interactive editor — scope is generation from markdown/issue/prompt, not WYSIWYG.
- Presenter mode (speaker notes + second screen) — not in v0.1.
- Remote-controlled decks — not in v0.1.
- Animations beyond the VE base set (fade + translateY + scale + staggered `--i` reveals).

## Complexity

**Tier: F-lite** — clear scope, single domain (forge plugin), no new cross-cutting architecture. Lifts well-defined patterns from Visual Explainer's `slide-patterns.md` and adapts them to forge conventions.

Signals observed:
- Single domain: `plugins/forge/` only — SKILL.md + one new `references/slide-patterns.md` + `references/slide-templates/{css,js,html}` + 1 demo deck.
- Clear scope: 10 slide types + 6 aesthetic presets + base CSS/JS explicitly enumerated in the issue.
- No architectural unknowns: reuses existing aesthetic system, fgraph primitives, Mermaid init, 4-phase pipeline.
- Reference implementation exists: VE's `slide-patterns.md` (1,406 lines) is the source to adapt.
- No new dependencies, no runtime changes, no supervisor-visible side effects.

## Decisions (from framing interview)

- **Aesthetic coverage (v0.1):** tune all 6 aesthetics (`blueprint`, `caveman`, `editorial`, `lyra`, `roxabi`, `terminal`) for slide mode. Consistent coverage over partial polish.
- **Content sources (v0.1):** accept all three inputs — issue `#N`, markdown file, free prompt — with precedence matching `forge-epic`.
- **PDF export:** deferred (see Out of Scope).
