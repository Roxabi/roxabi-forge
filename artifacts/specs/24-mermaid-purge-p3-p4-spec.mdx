## Goal

Delete all remaining Mermaid source files, remove stale decision rules from both CLAUDE.md files, and wire a blocking grep guard that prevents re-introduction.

## Users

- **Primary:** Mickael (forge author) — gains a clean codebase with no Mermaid vestiges; the blocking guard enforces the new default.
- **Secondary:** downstream artifact consumers — all outputs offline-safe, no CDN dependency.
- **Tertiary:** future forge contributors — CI guard blocks Mermaid additions, making the architectural decision visible.

## Expected Behavior

**Phase 3 (Delete):**

An author opens `plugins/forge/references/graph-templates/` and finds only the 6 native fgraph templates (`gantt.html`, `pie.html`, `er.html`, `sequence.html`, `state.html`, `dep-graph.html`). The Mermaid legacy files (`*-mermaid.html`), the `mermaid-init.js` runtime, and the `mermaid-guide.md` reference doc are gone. The `graph-templates/README.md` shows no Mermaid rows in any table. Both `~/.claude/CLAUDE.md` and `~/projects/CLAUDE.md` have the "> 8 nodes → Mermaid" decision-rule bullet removed.

**Phase 4 (Guard):**

A contributor attempts to add `import mermaid` to any file under `plugins/forge/` or the three render scripts. The pre-commit hook (`lefthook.yml`) runs `grep -rn '\bmermaid\b'` on the guarded paths and exits non-zero, blocking the commit with a message referencing #21. The CI workflow mirrors this check for PRs that bypass the local hook.

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Grep false positive (e.g., "mermaid" in prose comment) | Word-boundary `\bmermaid\b` matches whole words only; prose like "Mermaid purge" has capital M, exempt. If lowercase "mermaid" appears in a comment, it's a true match — author must rephrase. |
| CI guard fails transiently | CI step runs after lint; if grep itself fails (exit != 0 && no matches), it's a CI infrastructure issue, not a guard violation. Guard only fails when grep outputs matches. |
| Native templates accidentally deleted | Slice 8 deletes only `*-mermaid.html` files; native templates (`gantt.html`, `pie.html`, `er.html`) untouched. If accidentally deleted, restore from git history (they exist from P1). |
| Outside-repo CLAUDE.md not updated | PR body includes mandatory post-merge checklist; Mickael must verify before closing #24. No automated verification possible for files outside repo. |
