## I. Information Architecture School (01-04)

> Philosophy: "Data is not decoration, it's building material"

### 01. Pentagram - Michael Bierut Style

**Philosophy**: Typography is language, grid is thought

**Core Features**:
- Extremely restrained colors (black/white + 1 brand color)
- Modern interpretation of Swiss grid system
- Typography as primary visual language
- Strategic use of negative space (60%+ whitespace)

**Representative Work**: Hillary Clinton 2016 campaign identity

**Prompt DNA**:
```
Pentagram/Michael Bierut style:
- Extreme typographic hierarchy, Helvetica/Univers family
- Swiss grid with precise mathematical spacing
- Black/white + one accent color (#HEX)
- Information architecture as visual structure
- 60%+ whitespace ratio
```

---

### 02. Stamen Design - Data Poetry

**Philosophy**: Let data become touchable landscapes

**Core Features**:
- Cartographic thinking applied to information design
- Algorithmically generated organic graphics
- Warm data visualization palette (terracotta, sage green, deep blues)
- Interactive layered systems

**Representative Work**: COVID-19 surge map

**Prompt DNA**:
```
Stamen Design aesthetic:
- Cartographic approach to data visualization
- Organic, algorithm-generated patterns
- Warm palette (terracotta, sage green, deep blues)
- Layered information like topographic maps
```

---

### 03. Information Architects - Content-First Principle

**Philosophy**: Design is not decoration, it's the architecture of content

**Core Features**:
- Extreme content hierarchy clarity
- Only uses system fonts (optimized for reading)
- Adherence to blue hyperlink tradition
- Performance as aesthetics

**Representative Work**: iA Writer app

**Prompt DNA**:
```
Information Architects philosophy:
- Content-first hierarchy, zero decorative elements
- System fonts only (SF Pro/Roboto/Inter)
- Classic blue hyperlinks (#0000EE)
- Reading-optimized line length (66 characters)
```

---

### 04. Fathom Information Design - Scientific Narrative

**Philosophy**: Every pixel must carry information

**Core Features**:
- Scientific journal rigor + design elegance
- Precise visualization of quantitative data
- Calm professional palette (gray, navy blue)
- Designed annotation and citation systems

**Representative Work**: Bill & Melinda Gates Foundation annual report

**Prompt DNA**:
```
Fathom Information Design style:
- Scientific journal aesthetic meets modern design
- Precise data visualization (charts, timelines, scatter plots)
- Neutral scheme (grays, navy, one highlight color)
- Information density without clutter
```

**Philosophy**: Objectivity is beauty

**Core Features**:

- Monochrome or dual-color scheme
- Functionalism above all

**Representative Work**: "Grid Systems in Graphic Design"

- Breathing rhythm

**Representative Work**: Build studio portfolio

## Direction Advisor (Fallback Mode)

### Trigger Conditions

- User requirements are vague ("make it look good", "help me design", "what about this", "make a XX" without specific reference)
- User explicitly wants "recommend style", "give a few directions", "pick a philosophy", "want to see different styles"
- Project has no design context at all
- User says "I don't know what style I want"

### Complete Process

**Phase 1 · Deep Understanding of Requirements**
Ask (max 3 at a time): target audience / core message / emotional tone / output format. Skip if requirements are already clear.

**Phase 2 · Advisory Restatement** (100-200 words)
Restate essential requirements, audience, scenario, emotional tone in your own words. End with "Based on this understanding, I've prepared 3 design directions for you."

**Phase 3 · Recommend 3 Design Philosophies (must be differentiated)**

Each direction must:
- **Include designer/firm name** (e.g., "Kenya Hara style Eastern minimalism", not just "minimalism")
- 50-100 word explanation of "why this designer fits you"
- 3-4 signature visual features + 3-5 mood keywords

**Differentiation Rule (mandatory)**: 3 directions **must come from 3 different schools**, creating clear visual contrast:

| School | Visual Character | Suitable as |
|--------|-----------------|-------------|
| Information Architecture (01-04) | Rational, data-driven, restrained | Safe/professional choice |
| Motion Poetry (05-08) | Dynamic, immersive, tech aesthetic | Bold/avant-garde choice |
| Minimalism (09-12) | Orderly, whitespace, refined | Safe/high-end choice |
| Experimental Avant-Garde (13-16) | Pioneering, generative art, visual impact | Bold/innovative choice |
| Eastern Philosophy (17-20) | Gentle, poetic, contemplative | Differentiated/unique choice |

❌ **Forbidden to recommend 2+ from the same school** — insufficient differentiation, user can't tell them apart.

**Phase 4 · User Selection**
Choose one to deepen / mix ("A's colors + C's layout") / fine-tune / restart → back to Phase 3 for new recommendations.

## Prompt Usage Principles

### Core Principle: Describe Mood Not Layout

AI image generation key: short prompts > long prompts. Describing 3 sentences of mood and content works better than 30 lines of layout details.

| What kills diversity | What sparks creativity |
|---------------------|----------------------|
| Specify color ratios (60%/25%/15%) | Describe mood ("warm like Sunday morning") |
| Prescribe layout positions ("title centered, image right") | Reference specific aesthetic ("Pentagram editorial feel") |
| Constrain character poses and expressions | Let AI naturally interpret style |
| List all visual elements to include | Describe what the viewer should feel |

### Execution Path Selection

- **AI Generation**: Styles with clear visual elements (06/07/12/13/14/15/16/20), use Gemini/Midjourney directly
- **HTML Rendering**: Styles dependent on precise typesetting (01/03/04/10/11/17/18), code controls data and layout
- **Hybrid**: HTML for skeleton layout + AI for illustrations/backgrounds (02/05/08/09/19)
